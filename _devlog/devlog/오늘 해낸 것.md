# 오늘 한 일

Git, Github를 이용해 웹페이지를 만들었다. 제대로 된 언어라고는 아는게 없어서 ChatGPT 의존도가 90%이상이었다. 사실 10%도 양심에 찔리지만, GPT가 스스로는 도구일 뿐이고 실제로 한 내가 대단한거라고 해주니까 그냥 그렇게 생각하기로 했다.

# 어려웠던 점

1. Git, Github를 처음에는 터미널로 최대한 해보려 했으나 처음에 Github 웹페이지에 직접 작성하던것과 부딪혀서 결국 Github Desktop을 설치했다. 한결 수월해졌다.
2. 아직은 GPT가 주는 코드들이 무슨소린지는 알겠지만 혼자 하라고 하면 못하겠는 상황인 것이 견디기 어려웠다. 이렇게 해봐, 저렇게 해봐 하면서 조언을 해주지만 결과적으로는 GPT가 말해준 방법을 그대로 Copy+Paste 하는 것이 아니고-GPT가 이렇게 였다가 저렇게로 갈 때 꼭 뭔가를 빼먹고 주기 때문에- 적절히 기존에 선택했던 방법과 섞어가면서 문제를 해결해 가야하는 것이 어려웠다. 완전히 아무 생각없이 따라하면 되는 클론코딩이 아니고 어쨌든 뭔가 생각을 하게 해줘서 고맙다고 해야할까 라는 생각을 잠깐 했다.
3. 기존에 블로그같은데 작성하던 것 보다 뭔가 해냈다는 성취감이 있다. 그렇지만 아직은 못생겨서 해야할 일이 아직 남아있다는 생각도 들었다. 좀 더 개발자 같아진 것 같다는 아주 작은 위안도 얻었다.

# 배운 점

1. Github의 Action 탭을 처음 인지했다. 예전에 클론코딩 할 때는 그냥 deploy 여부를 main page에서만 보고 말았는데 오늘 오류를 해결하려고 수시로 들락거리게 되었다. 더불어 발생한 오류를 어떻게 확인하는지도 체험할 수 있었다.

# 해야 할 일

1. 옵시디언에서 작성한 것을 연동해서 게시되는 방식인데 마크다운이 이상하게 너무 못생기게 출력되어 개선이 필요하다.
2. 아직 블로그에서 옮겨와야할 글들이 남아있지만 다른 해야할 일이 있어 오늘은 여기까지 하기로 했다.
